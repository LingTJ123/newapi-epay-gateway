# 安全说明

## 已实现的控制

- 所有应用密钥、数据库密码和管理员密码均从环境或只读证书卷加载
- 启动时使用 Zod 校验关键配置和证书可读性，失败即退出
- EPay 签名比较使用定时安全比较；支付宝通知使用官方 SDK 验签
- 金额使用严格十进制解析并以整数分持久化，拒绝浮点、科学计数法和两位以上小数
- `notify_url` 与 `return_url` 必须严格等于白名单，阻止 SSRF
- 订单号、商品名、长度和控制字符均有限制
- 支付宝通知同时校验 APPID、seller_id、订单号、金额、交易状态和交易号
- 数据库乐观锁与 NewAPI 通知抢占锁避免并发重复回调
- Helmet、安全 body 大小、限流、可信代理层级、请求超时
- Pino 对 Authorization、签名、KEY、私钥和密码字段脱敏
- 网关容器以非 root 用户运行，只读根文件系统，丢弃 Linux capabilities
- 3100 只映射主机 `127.0.0.1`；数据库不映射主机端口

## 运维要求

- `.env` 权限设为 600，`secrets/` 权限设为 700，私钥设为 600
- 只允许通过 HTTPS 访问管理页；管理员密码至少 12 位且不得复用
- 不要在工单、截图、聊天或日志中发送私钥、EPay KEY、数据库密码或完整买家账号
- 定期轮换管理员密码和数据库密码；轮换 EPay KEY 时必须安排 NewAPI 与网关同步切换窗口
- 监控证书有效期、服务器磁盘、数据库备份和连续回调失败
- 每次升级前运行 `npm audit --omit=dev`、测试和数据库备份

## 幂等边界

网关会在单进程和多副本并发通知下通过数据库抢占锁保证同一时刻只有一个 NewAPI 回调。若进程在“NewAPI 已入账、网关尚未来得及标记成功”的极小窗口崩溃，恢复后可能使用同一 `out_trade_no` 重放通知。跨两个独立系统无法在没有分布式事务的情况下保证绝对 exactly-once；NewAPI 必须按订单号幂等处理 EPay 通知。上线演练应验证重复通知不会重复加额度。

## 事件响应

怀疑密钥泄漏时：

1. 暂停支付入口或在 1Panel 临时禁用 `/epay` 代理。
2. 在支付宝开放平台更换应用密钥/证书并保存旧订单证据。
3. 轮换 EPay KEY、数据库密码和管理员密码。
4. 审查网关日志、`payment_events`、`notify_attempts` 和 NewAPI 充值记录。
5. 重新构建容器并完成小额验收后恢复入口。

安全问题请通过仓库的私密安全报告渠道提交，不要创建包含密钥或交易个人信息的公开 Issue。
