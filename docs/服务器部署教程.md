# NewAPI EPay 网关服务器部署教程

本文使用保留示例域名 `newapi.example.com` 演示以下部署：NewAPI 与支付网关位于同一台 Ubuntu 服务器；1Panel 管理 OpenResty；NewAPI 监听 `127.0.0.1:3000`；网关通过 `https://newapi.example.com/epay` 访问并监听主机的 `127.0.0.1:3100`。实际部署时必须把所有 `newapi.example.com` 替换为自己的 NewAPI 公网域名。教程不要求额外域名，也不修改 NewAPI 源码。

> 支付属于高风险业务。正式收款前必须先完成沙箱和小额真机测试，并确认支付宝应用、签约主体、APPID、seller_id 和证书属于同一主体/应用。

## 1. 最终请求路径

```text
浏览器 / NewAPI
  → https://newapi.example.com/epay/submit.php
  → 1Panel OpenResty（保留 /epay 前缀）
  → http://127.0.0.1:3100/epay/submit.php
  → 本网关
  → 支付宝电脑网站收银台

支付宝异步通知
  → https://newapi.example.com/epay/alipay/notify
  → 网关验签、校验金额并持久化
  → https://newapi.example.com/api/user/epay/notify
  → NewAPI 增加额度
```

网关同时注册有前缀和无前缀路由，因此 1Panel 不需要 Rewrite。

## 2. 上线前清单

准备以下信息：

- 支付宝应用 APPID（支付宝开放平台分配的 10–32 位数字）
- 支付宝商户 seller_id/PID，通常以 `2088` 开头；它不是 APPID
- PKCS8 应用私钥
- 应用公钥证书 `appCertPublicKey.crt`
- 支付宝公钥证书 `alipayCertPublicKey_RSA2.crt`
- 支付宝根证书 `alipayRootCert.crt`
- NewAPI 与网关共用的 EPay PID 和 64 位十六进制 EPay KEY
- 两个不同的 MariaDB 强密码
- 一个至少 12 位的管理员密码

服务器应满足：

```bash
docker --version
docker compose version
git --version
openssl version
timedatectl status
```

确保系统时间同步。支付签名和证书验证依赖正确时间：

```bash
sudo timedatectl set-ntp true
```

## 3. 支付宝开放平台配置

官方参考：[电脑网站支付快速接入](https://opendocs.alipay.com/open/270/105899)、[`alipay.trade.page.pay`](https://opendocs.alipay.com/open/028r8t)、[异步通知说明](https://opendocs.alipay.com/open/204/105301)。界面文案可能随平台更新略有变化，以应用控制台实际显示为准。

### 3.1 创建并上线应用

1. 登录 [支付宝开放平台](https://open.alipay.com/)，进入“开发者中心”。
2. 创建“自用型网页应用”，主体应与收款支付宝商户主体一致。
3. 在应用产品/能力中添加“电脑网站支付”。若页面要求签约，完成商家签约和资质审核。
4. 确认应用拥有 `alipay.trade.page.pay`、`alipay.trade.query` 和 `alipay.trade.close` 所需能力。
5. 开发配置和能力审核完成后提交应用上线；正式 APPID 必须处于可用状态。

### 3.2 配置公钥证书模式

1. 在应用的“开发设置/接口加签方式”中选择“公钥证书”。
2. 使用支付宝开放平台密钥工具生成 RSA2 2048 位密钥和 CSR，或上传已经生成且与当前应用私钥匹配的 CSR。
3. 上传 CSR 后下载这三个证书：
   - 应用公钥证书
   - 支付宝公钥证书
   - 支付宝根证书
4. 不要将“应用公钥文本”当作“应用公钥证书”；证书模式必须使用 `.crt` 文件。
5. 本项目不需要 AES 内容加密密钥。

最终在本地准备并统一命名：

```text
app_private_key.pem
appCertPublicKey.crt
alipayCertPublicKey_RSA2.crt
alipayRootCert.crt
```

### 3.3 转换应用私钥

如果私钥文件只有以 `MIIEv...` 开头的正文，没有 PEM 头尾，先在离线可信机器上转换。假设原文件叫 `应用私钥RSA2048-敏感数据，请妥善保管.txt`：

```bash
{
  echo '-----BEGIN PRIVATE KEY-----'
  tr -d '\r\n ' < '应用私钥RSA2048-敏感数据，请妥善保管.txt' | fold -w 64
  echo '-----END PRIVATE KEY-----'
} > app_private_key.pem
```

验证格式，不应要求输入密码，也不应报错：

```bash
openssl pkey -in app_private_key.pem -check -noout
```

### 3.4 校验私钥与应用公钥证书匹配

```bash
openssl pkey -in app_private_key.pem -pubout -outform DER | sha256sum
openssl x509 -in appCertPublicKey.crt -pubkey -noout \
  | openssl pkey -pubin -outform DER \
  | sha256sum
```

两个 SHA-256 必须完全一致。不一致时不要部署，应回到支付宝密钥工具确认 CSR 使用的是哪一把私钥。

检查证书主体和有效期：

```bash
openssl x509 -in appCertPublicKey.crt -noout -subject -issuer -dates
openssl x509 -in alipayCertPublicKey_RSA2.crt -noout -subject -issuer -dates
```

### 3.5 支付宝后台哪些项不用填

保持以下项目空白即可：

- 应用网关
- OAuth 授权回调地址
- AES 内容加密密钥

网关会在每笔 `alipay.trade.page.pay` 请求中提交：

```text
notify_url=https://newapi.example.com/epay/alipay/notify
return_url=https://newapi.example.com/epay/alipay/return
```

支付宝的异步通知以程序传入的 `notify_url` 为准，不依赖“应用网关”。建议在应用安全设置中配置服务器固定公网出口 IP 白名单：

```bash
curl -4 https://api.ipify.org ; echo
```

### 3.6 沙箱注意事项

沙箱联调时必须同时使用沙箱 APPID、沙箱密钥/证书、沙箱买家账号和沙箱网关，不能和正式环境参数混用：

```env
ALIPAY_GATEWAY=https://openapi-sandbox.dl.alipaydev.com/gateway.do
```

完成沙箱后切换正式环境时，重新核对 APPID、seller_id、四个证书/私钥文件和网关地址。

## 4. 安装网关

### 4.1 备份旧简化版

如果 `/opt/newapi-alipay-gateway` 已存在：

```bash
sudo cp -a /opt/newapi-alipay-gateway \
  "/opt/newapi-alipay-gateway.backup.$(date +%Y%m%d%H%M%S)"
```

保留旧 `.env` 和 `secrets/` 作为迁移参考，但不要直接覆盖新模板。

### 4.2 拉取代码

```bash
sudo mkdir -p /opt/newapi-alipay-gateway
sudo chown -R "$USER":"$USER" /opt/newapi-alipay-gateway
git clone https://github.com/your-github-account/newapi-epay-gateway.git /opt/newapi-alipay-gateway
cd /opt/newapi-alipay-gateway
```

如果目录已经是本仓库：

```bash
cd /opt/newapi-alipay-gateway
git pull --ff-only
```

### 4.3 放置证书

```bash
mkdir -p secrets
chmod 700 secrets
```

通过 1Panel 文件管理器或 `scp` 上传四个文件到 `/opt/newapi-alipay-gateway/secrets/`。随后执行：

```bash
chmod 600 secrets/app_private_key.pem
chmod 644 secrets/appCertPublicKey.crt \
  secrets/alipayCertPublicKey_RSA2.crt \
  secrets/alipayRootCert.crt
```

不要把证书或私钥上传到 GitHub。`.gitignore` 和打包脚本会排除 `secrets/` 中的实际文件，但仍应在提交前运行 `git status` 检查。

## 5. 配置 `.env`

`.env` 同时为网关容器和 MariaDB 容器提供配置。请只在服务器项目目录中保存它，不要提交到 Git，也不要通过聊天、工单或截图公开其内容。

### 5.1 创建文件

```bash
cd /opt/newapi-alipay-gateway
cp .env.example .env
chmod 600 .env
```

书写规则：

- 每行使用 `变量名=值`，等号两侧不要加空格。
- 不要保留 `<...>`、`replace_with...`、`xxxxxxxx` 等占位文本。
- URL 使用自己的公网 HTTPS 地址，不要在末尾加 `/`；本文以 `https://newapi.example.com/epay` 为例，不能原样用于生产。
- 除 `ADMIN_PASSWORD_HASH` 外，按下面模板填写时不需要额外引号。
- `ADMIN_PASSWORD_HASH` 必须用单引号包住完整 bcrypt 哈希，以防 Docker Compose 把哈希中的 `$` 当成变量插值。
- 保存为 UTF-8 纯文本。不要使用全角标点、中文引号或在值后面追加行内注释。

### 5.2 先生成四个秘密值

前三个值必须分别生成，不能相互复用：

```bash
openssl rand -hex 32   # EPAY_KEY
openssl rand -hex 24   # DB_PASSWORD
openssl rand -hex 24   # DB_ROOT_PASSWORD
```

命令每执行一次会输出一个新值：

- 第一条输出 64 位十六进制字符串，填入 `EPAY_KEY`。
- 第二条输出 48 位十六进制字符串，填入 `DB_PASSWORD`，并在 `DATABASE_URL` 的密码位置原样再写一次。
- 第三条输出另一个 48 位十六进制字符串，填入 `DB_ROOT_PASSWORD`。

例如，假设第二条命令输出的是 `你的48位十六进制数据库密码`，则两行必须是：

```env
DB_PASSWORD=你的48位十六进制数据库密码
DATABASE_URL=mysql://epay_gateway:你的48位十六进制数据库密码@db:3306/epay_gateway
```

上例中的中文只是位置说明，不能原样写入 `.env`。使用推荐的十六进制密码时不需要 URL 编码。若自行设置的数据库密码含有 `@`、`:`、`/`、`#`、`?`、`%` 等 URL 保留字符，`DATABASE_URL` 中的密码部分必须做百分号编码；为了避免填错，生产部署建议直接使用上面的 `openssl` 命令。

管理员密码不能直接写进 `.env`，需要先生成 bcrypt 哈希。在一台安装了 Node.js 22 的可信电脑或服务器项目目录执行：

```bash
npm ci
npm run admin:hash -- '替换为至少12位的管理员密码'
```

复制命令输出的整行 60 字符哈希。例如输出形状为 `$2b$12$...`，写法必须是：

```env
ADMIN_PASSWORD_HASH='$2b$12$这里应是完整的60字符哈希'
```

单引号只用于保护 `$`，Docker Compose 传给程序的是不带引号的哈希值。管理员密码建议至少 12 位，并混合大小写字母、数字和符号。请使用项目自带命令生成，避免在线 bcrypt 网站或来源不明的第三方 CLI 接触明文密码。若服务器没有 Node.js，可以先保留占位值，完成镜像构建后使用容器生成：

```bash
docker compose build gateway
docker compose run --rm --no-deps --entrypoint node gateway \
  --input-type=module \
  -e 'import bcrypt from "bcryptjs"; console.log(await bcrypt.hash(process.argv[1], 12));' \
  '替换为至少12位的管理员密码'
```

### 5.3 可直接填写的完整 `.env` 模板

使用 `nano .env` 或 1Panel 文件编辑器打开文件，将其整理为下面内容。标有 `REPLACE_` 的值必须替换；其他值适用于本文的域名和 Docker Compose 部署方式：

```env
NODE_ENV=production
PORT=3100
LOG_LEVEL=info
TRUST_PROXY=true

EPAY_PID=10001
EPAY_KEY=REPLACE_WITH_64_HEX_FROM_OPENSSL
EPAY_TYPE=alipay
MIN_AMOUNT=0.01
MAX_AMOUNT=100.00

GATEWAY_PUBLIC_URL=https://newapi.example.com/epay
NEWAPI_NOTIFY_URL=https://newapi.example.com/api/user/epay/notify
NEWAPI_RETURN_URL=https://newapi.example.com/usage-logs
ALLOWED_NEWAPI_NOTIFY_URL=https://newapi.example.com/api/user/epay/notify
ALLOWED_NEWAPI_RETURN_URL=https://newapi.example.com/usage-logs

ALIPAY_APP_ID=REPLACE_WITH_ALIPAY_APP_ID
ALIPAY_SELLER_ID=REPLACE_WITH_2088_MERCHANT_PID
ALIPAY_GATEWAY=https://openapi.alipay.com/gateway.do
ALIPAY_KEY_TYPE=PKCS8
ALIPAY_PRIVATE_KEY_PATH=/run/secrets/app_private_key.pem
ALIPAY_APP_CERT_PATH=/run/secrets/appCertPublicKey.crt
ALIPAY_PUBLIC_CERT_PATH=/run/secrets/alipayCertPublicKey_RSA2.crt
ALIPAY_ROOT_CERT_PATH=/run/secrets/alipayRootCert.crt

DB_PASSWORD=REPLACE_WITH_DB_PASSWORD
DB_ROOT_PASSWORD=REPLACE_WITH_ANOTHER_ROOT_PASSWORD
DATABASE_URL=mysql://epay_gateway:REPLACE_WITH_SAME_DB_PASSWORD@db:3306/epay_gateway

NOTIFY_RETRY_ENABLED=true
NOTIFY_RETRY_INTERVAL_SECONDS=60
NOTIFY_RETRY_MAX_COUNT=20
PENDING_QUERY_INTERVAL_SECONDS=120
ORDER_EXPIRE_MINUTES=30

ADMIN_ENABLED=true
ADMIN_USERNAME=admin
ADMIN_PASSWORD_HASH='$2b$12$REPLACE_WITH_COMPLETE_60_CHARACTER_BCRYPT_HASH'

RATE_LIMIT_ENABLED=true
REQUEST_TIMEOUT_MS=10000
```

必须替换的具体值：

1. `EPAY_KEY`：替换为 `openssl rand -hex 32` 的第一条输出。
2. `ALIPAY_SELLER_ID`：替换为支付宝商户账号对应的 2088 开头 PID，不是应用 APPID。
3. `DB_PASSWORD`：替换为第二条 `openssl` 输出；`DATABASE_URL` 中 `REPLACE_WITH_SAME_DB_PASSWORD` 也替换为完全相同的值。
4. `DB_ROOT_PASSWORD`：替换为第三条 `openssl` 输出，必须与 `DB_PASSWORD` 不同。
5. `ADMIN_PASSWORD_HASH`：替换为项目命令生成的完整 bcrypt 输出，并保留外层单引号。
6. `ALIPAY_APP_ID`：替换为支付宝开放平台为当前应用分配的 APPID；它必须与上传的应用公钥证书和应用私钥属于同一个应用。

### 5.4 每个变量应当怎么填

| 变量 | 本项目应填的值 | 值的来源和注意事项 |
| --- | --- | --- |
| `NODE_ENV` | `production` | 生产服务器固定使用该值。 |
| `PORT` | `3100` | 网关容器监听端口。Compose 只把它映射到服务器本机 `127.0.0.1:3100`，公网由 OpenResty 反向代理。 |
| `LOG_LEVEL` | `info` | 生产环境推荐值。排障时可短暂改成 `debug`，完成后恢复，避免日志过多。 |
| `TRUST_PROXY` | `true` | 本教程前面只有一层可信 OpenResty 代理，因此使用 `true`。不要绕过代理把 3100 端口开放到公网。 |
| `EPAY_PID` | `10001` | 网关自定义商户号，不是支付宝 PID。NewAPI 易支付设置中的“商户 ID”必须也填 `10001`。可改为 1–32 位字母、数字、下划线或连字符，但两端必须完全一致。 |
| `EPAY_KEY` | 64 位十六进制随机值 | 由 `openssl rand -hex 32` 生成。NewAPI 易支付设置中的“商户密钥”必须填同一个值。它不是支付宝应用私钥。 |
| `EPAY_TYPE` | `alipay` | 当前网关只支持支付宝，固定不要改。 |
| `MIN_AMOUNT` | `0.01` | 单笔最小金额，单位为元，最多两位小数。NewAPI 允许的最小充值金额不要低于此值。 |
| `MAX_AMOUNT` | `100.00` | 单笔最大金额，单位为元，最多两位小数；可按业务风控调整，但不得小于 `MIN_AMOUNT`。 |
| `GATEWAY_PUBLIC_URL` | `https://newapi.example.com/epay` | 用户和 NewAPI 访问网关的公网根地址，必须是 HTTPS，包含 `/epay`，末尾不要加 `/`；把示例域名替换为自己的域名。 |
| `NEWAPI_NOTIFY_URL` | `https://newapi.example.com/api/user/epay/notify` | 网关通知 NewAPI 支付结果的服务端地址。使用同域部署时把示例域名替换为自己的域名。 |
| `NEWAPI_RETURN_URL` | `https://newapi.example.com/usage-logs` | 当前 NewAPI 会把“系统设置 → 服务器地址”去掉末尾 `/` 后拼接 `/usage-logs`；应以自己的 NewAPI 服务器地址按相同规则生成。 |
| `ALLOWED_NEWAPI_NOTIFY_URL` | 与 `NEWAPI_NOTIFY_URL` 完全一致 | 回调白名单，用于阻止请求伪造。包括协议、域名、端口、路径和末尾斜杠都必须一致。 |
| `ALLOWED_NEWAPI_RETURN_URL` | 与 `NEWAPI_RETURN_URL` 完全一致 | 返回地址白名单，同样要求逐字符一致。 |
| `ALIPAY_APP_ID` | 实际 APPID | 在支付宝开放平台进入应用，在应用概览查看。必须与 `appCertPublicKey.crt` 和应用私钥属于同一个应用。 |
| `ALIPAY_SELLER_ID` | 实际的 2088 开头商户 PID | 在支付宝商家平台/开放平台账号信息中查看“合作伙伴身份 PID”或“商户 PID”。不要填 APPID、登录邮箱或收款账号。 |
| `ALIPAY_GATEWAY` | `https://openapi.alipay.com/gateway.do` | 支付宝正式环境网关。只有沙箱联调时才改为沙箱网关，并同时换成沙箱 APPID、沙箱账号和配套密钥/证书。 |
| `ALIPAY_KEY_TYPE` | `PKCS8` | 本项目固定使用 PKCS#8 应用私钥，不要改。 |
| 四个 `ALIPAY_*_PATH` | 保持模板值 | 这是容器内路径，不是服务器上的绝对路径。Compose 会把宿主机 `./secrets` 只读挂载到 `/run/secrets`。 |
| `DB_PASSWORD` | 48 位十六进制随机值 | MariaDB 业务用户 `epay_gateway` 的密码，由第二条 `openssl` 命令生成。 |
| `DB_ROOT_PASSWORD` | 另一个 48 位十六进制随机值 | MariaDB root 密码，由第三条命令独立生成；日常网关不会使用它。 |
| `DATABASE_URL` | `mysql://epay_gateway:业务密码@db:3306/epay_gateway` | 用户名、主机名、端口和库名保持模板值。中间密码必须与 `DB_PASSWORD` 完全一致；不要写成 `${DB_PASSWORD}`，直接写实际值。 |
| `NOTIFY_RETRY_ENABLED` | `true` | NewAPI 暂时无法接收通知时自动重试。生产环境建议开启。 |
| `NOTIFY_RETRY_INTERVAL_SECONDS` | `60` | 通知重试间隔秒数，允许值不小于 10。 |
| `NOTIFY_RETRY_MAX_COUNT` | `20` | 单笔订单最多通知次数，允许 1–100。 |
| `PENDING_QUERY_INTERVAL_SECONDS` | `120` | 后台向支付宝查询待支付订单的间隔秒数，允许值不小于 30。 |
| `ORDER_EXPIRE_MINUTES` | `30` | 本地订单过期时间，允许 1–1440 分钟。 |
| `ADMIN_ENABLED` | `true` | 开启 `/epay/admin` 只读管理页。若明确不需要管理页，可改为 `false`，此时 `ADMIN_PASSWORD_HASH` 可留空。 |
| `ADMIN_USERNAME` | 建议改为非默认用户名 | 管理页 Basic Auth 用户名。不要与服务器 SSH 用户名或数据库用户名复用。 |
| `ADMIN_PASSWORD_HASH` | 单引号包裹的 60 字符 bcrypt 哈希 | 只能填项目命令输出的哈希，不能填管理员明文密码。启动时会校验完整格式。 |
| `RATE_LIMIT_ENABLED` | `true` | 启用接口限流，生产环境建议开启。 |
| `REQUEST_TIMEOUT_MS` | `10000` | 对外请求超时，单位毫秒，允许 1000–60000；一般保持 10 秒。 |

### 5.5 正式环境与沙箱不能混用

正式环境的 APPID、应用私钥、三份支付宝证书、商户 PID 和支付宝网关必须属于同一套正式应用。沙箱联调则必须整套替换为沙箱资料，不能只替换 `ALIPAY_GATEWAY`。否则常见结果是验签失败、应用证书序列号不匹配或收款商户不一致。

| 环境 | `ALIPAY_GATEWAY` | 账号、APPID、密钥和证书 |
| --- | --- | --- |
| 正式 | `https://openapi.alipay.com/gateway.do` | 全部使用正式应用资料 |
| 沙箱 | `https://openapi-sandbox.dl.alipaydev.com/gateway.do` | 全部使用同一沙箱应用和沙箱买卖家资料 |

### 5.6 保存后进行无泄密检查

先检查是否还有占位文本；以下命令只显示占位行，不会主动打印已正确填写的秘密值：

```bash
grep -nE 'REPLACE_|replace_with|xxxxxxxx|<[^>]+>' .env
```

没有输出才表示占位文本已经清理。然后在当前终端加载配置并做一致性检查，命令只输出检查结论，不输出密钥本身：

```bash
set -a
. ./.env
set +a

printf '%s' "$EPAY_KEY" | grep -Eq '^[0-9a-f]{64}$' \
  && echo "EPAY_KEY 格式正确" \
  || { echo "EPAY_KEY 必须是 64 位小写十六进制字符"; exit 1; }

test "$DB_PASSWORD" != "$DB_ROOT_PASSWORD" \
  && echo "两个数据库密码不同" \
  || { echo "两个数据库密码不能相同"; exit 1; }

test "$NEWAPI_NOTIFY_URL" = "$ALLOWED_NEWAPI_NOTIFY_URL" \
  && echo "通知地址白名单一致" \
  || { echo "通知地址不一致"; exit 1; }

test "$NEWAPI_RETURN_URL" = "$ALLOWED_NEWAPI_RETURN_URL" \
  && echo "返回地址白名单一致" \
  || { echo "返回地址不一致"; exit 1; }

case "$DATABASE_URL" in
  "mysql://epay_gateway:${DB_PASSWORD}@db:3306/epay_gateway")
    echo "DATABASE_URL 与 DB_PASSWORD 一致"
    ;;
  *)
    echo "DATABASE_URL 中的密码或连接信息不正确"
    exit 1
    ;;
esac
```

再确认四个证书文件存在且可读：

```bash
for file in \
  secrets/app_private_key.pem \
  secrets/appCertPublicKey.crt \
  secrets/alipayCertPublicKey_RSA2.crt \
  secrets/alipayRootCert.crt
do
  test -r "$file" || { echo "文件不存在或不可读：$file"; exit 1; }
done
echo "四个密钥/证书文件均可读"
```

最后让 Docker Compose 验证语法：

```bash
docker compose config --quiet
```

不要把 `docker compose config` 的完整输出发到公开渠道，因为展开结果包含数据库密码等秘密。启动时程序还会校验关键变量、bcrypt 哈希以及四个文件；任何一项不合格，网关都会退出并在日志中指出变量名，不会带着错误配置继续运行。

### 5.7 `.env` 与 NewAPI 设置的对应关系

配置 NewAPI 时，三项必须这样对应：

| NewAPI 易支付字段 | 应填写的值 | 对应 `.env` |
| --- | --- | --- |
| 易支付网关地址/接口地址 | `https://newapi.example.com/epay` | `GATEWAY_PUBLIC_URL` |
| 商户 ID | `10001` | `EPAY_PID` |
| 商户密钥 | 与 `.env` 中 64 位随机值完全相同 | `EPAY_KEY` |

`EPAY_KEY` 只在 NewAPI 与本网关之间共享；支付宝开放平台不填写它。支付宝应用私钥只保存为 `secrets/app_private_key.pem`，也不要复制到 NewAPI。

本机验证：

```bash
curl -i http://127.0.0.1:3100/healthz
```

## 7. 1Panel/OpenResty 配置

在 1Panel 中打开自己的 NewAPI 域名网站（本文示例为 `newapi.example.com`），添加或保留以下反向代理：

```text
前端路径：/epay
后端地址：http://127.0.0.1:3100
```

不添加 URI Rewrite，不把后端改成 `/`，也不需要新域名。1Panel 生成的配置类似：

```nginx
location /epay {
    proxy_pass http://127.0.0.1:3100;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
    proxy_http_version 1.1;
}
```

保存并重载 OpenResty 后验证公网路径：

```bash
curl -i https://newapi.example.com/epay/healthz
curl -i -X POST https://newapi.example.com/epay/submit.php
```

第一条必须为 200；第二条应为 400 业务错误。若返回 NewAPI 页面，说明 `/epay` 代理没有命中；若返回 502，检查容器和 `127.0.0.1:3100`；若返回 404，检查 1Panel 实际转发路径。

## 8. NewAPI 配置

进入 NewAPI 管理后台的支付/充值/EPay 设置。不同版本菜单名称可能略有差异，填写值如下：

| NewAPI 字段 | 填写值 |
|---|---|
| EPay 端点 | `https://newapi.example.com/epay` |
| 回调地址 | `https://newapi.example.com` |
| 易支付商户 ID | 与网关 `EPAY_PID` 完全一致，例如 `10001` |
| EPay 密钥 | 与网关 `EPAY_KEY` 完全一致 |

EPay 端点不能填写 `https://newapi.example.com/epay/submit.php`，因为 NewAPI 会自动追加 `/submit.php`。填错后会形成 `/submit.php/submit.php`。

支付方式配置：

```json
[
  {
    "color": "rgba(var(--semi-blue-5), 1)",
    "name": "支付宝",
    "type": "alipay",
    "min_topup": "1"
  }
]
```

首次联调建议：

```text
最低充值：1
充值价格：1
分组倍率：1
充值折扣：暂不配置
支付合规声明：确认
```

保存后重新打开配置页核对值。特别检查 EPay KEY 是否被浏览器自动填充功能改写。

## 9. 首次联调与验收

### 9.1 先沙箱

1. 将支付宝 APPID、证书、网关全部切换为同一套沙箱参数。
2. 在 NewAPI 用沙箱允许的最低金额创建充值。
3. 浏览器应进入支付宝沙箱收银台。
4. 支付后查看：

```bash
docker compose logs --since=10m gateway
docker compose exec db mariadb -u epay_gateway -p epay_gateway \
  -e 'SELECT out_trade_no,status,newapi_notify_status,amount_cents,alipay_trade_no FROM orders ORDER BY id DESC LIMIT 10;'
```

输入的是 `DB_PASSWORD`。正常最终状态为：

```text
status=COMPLETED
newapi_notify_status=SUCCESS
```

5. 在 NewAPI 检查充值记录和用户额度。

### 9.2 再正式小额

切换完整正式配置并重建：

```bash
docker compose up -d --build --force-recreate gateway
```

用 0.01 元或 1 元完成真实支付，具体最低金额需同时满足支付宝产品和 NewAPI 配置。核对：支付宝订单、本地订单、NewAPI 充值记录、用户额度四处金额一致。

### 9.3 故障恢复演练

在测试环境临时停止 NewAPI，随后完成支付：

1. 网关订单应保存为 `PAID`、通知状态为 `FAILED`。
2. 恢复 NewAPI。
3. 网关按 1、2、5、10、30、60 分钟阶梯自动重试。
4. 最终变为 `COMPLETED/SUCCESS`，额度只增加一次。

管理入口 `https://newapi.example.com/epay/admin` 可查询订单，并手动“重试通知”或“支付宝查单”。实际访问时替换为自己的域名。管理入口使用 HTTPS Basic Auth，浏览器会弹出用户名/密码框。

## 11. 常见故障

### `Cannot POST /epay/submit.php`

正在运行旧简化版镜像，或 1Panel 指向了错误端口。执行：

```bash
docker compose up -d --build --force-recreate gateway
curl -i http://127.0.0.1:3100/epay/submit.php
```

### 健康检查 502

```bash
docker compose ps
docker compose logs --tail=200 gateway
sudo ss -lntp | grep ':3100'
```

常见原因是 `.env` 校验失败、证书文件名错误、私钥权限错误或数据库尚未健康。

### 支付宝报签名错误

- APPID 与应用证书是否来自同一应用
- 应用私钥与应用公钥证书哈希是否一致
- `ALIPAY_KEY_TYPE=PKCS8`
- 正式/沙箱网关是否与 APPID 配套
- 服务器时间是否准确

### `return_url 不在白名单中`

当前 NewAPI 使用“系统设置 → 服务器地址”拼接 `/usage-logs` 作为充值完成返回页。以服务器地址 `https://newapi.example.com` 为例，`.env` 中必须同时填写：

```env
NEWAPI_RETURN_URL=https://newapi.example.com/usage-logs
ALLOWED_NEWAPI_RETURN_URL=https://newapi.example.com/usage-logs
```

修改后运行 `docker compose up -d --force-recreate gateway` 让容器重新读取 `.env`，无需重新构建镜像。若 NewAPI 的服务器地址不是该域名，应以 NewAPI 实际服务器地址为准；协议、域名、端口、路径和末尾斜杠必须逐字符一致。

### 支付成功但 NewAPI 未到账

先查订单状态和通知尝试：

```bash
docker compose exec db mariadb -u epay_gateway -p epay_gateway \
  -e 'SELECT out_trade_no,status,newapi_notify_status,newapi_notify_attempts,newapi_notify_last_error FROM orders ORDER BY id DESC LIMIT 10;'
```

确认 NewAPI 回调必须返回正文恰好为小写 `success`。仅 HTTP 200 不算成功。检查 NewAPI 的 EPay PID/KEY 与网关一致，并可在管理页手动重试。

### 支付宝通知反复重试

网关只有在 NewAPI 已成功入账后才向支付宝返回 `success`。这通常表示 NewAPI 回调失败。不要通过关闭验签或强制返回成功来掩盖问题，否则会导致已收款但未加额度。

### 管理页 401

重新生成 bcrypt 哈希并只重建网关：

```bash
npm run admin:hash -- '新的至少12位密码'
# 把输出完整写入 ADMIN_PASSWORD_HASH
docker compose up -d --force-recreate gateway
```
